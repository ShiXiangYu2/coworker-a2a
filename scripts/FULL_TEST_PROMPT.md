## 提示词 3：自动化全量测试

```
请执行 CoWorker 项目的自动化全量测试：

## 测试执行步骤

### 步骤 1：环境准备
```bash
# 检查 Node.js 版本
node --version

# 检查 npm 版本
npm --version

# 检查项目依赖
npm ls --depth=0
```

### 步骤 2：代码质量检查
```bash
# 运行 ESLint
npm run lint

# 运行 TypeScript 检查
npx tsc --noEmit
```

### 步骤 3：单元测试
```bash
# 运行所有测试
npm run test

# 运行测试并生成覆盖率报告
npm run test -- --coverage
```

### 步骤 4：集成测试
```bash
# 启动开发服务器
npm run dev &

# 等待服务器启动
sleep 5

# 测试 API 端点
curl -X POST http://localhost:3000/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message":"hello"}'

# 停止服务器
kill %1
```

### 步骤 5：数据库测试
```bash
# 检查数据库连接
npx prisma db push

# 生成 Prisma Client
npx prisma generate

# 检查数据库状态
npx prisma studio
```

### 步骤 6：业务闭环测试
```bash
# 运行端到端测试
npx tsx scripts/runtime-seed-sample-job.ts --taskId="e2e-test-$(date +%s)"
npx tsx scripts/runtime-run-once.ts --jobId="<jobId>" --workerId="worker-dev-1" --mode=dry_run
npx tsx scripts/runtime-verify-once.ts --jobId="<jobId>" --workerId="worker-dev-1"

## 使用方法

1. 复制上述任意提示词
2. 粘贴到 ChatHub 界面
3. 发送消息
4. 系统会自动执行测试
5. 查看测试结果

## 预期结果

系统应该能够：
- ✅ 自动执行所有测试步骤
- ✅ 生成详细的测试报告
- ✅ 识别问题并提供修复建议
- ✅ 评估业务闭环完整性

## 注意事项

1. 确保开发服务器已启动
2. 确保数据库已初始化
3. 测试过程中不要关闭开发服务器
4. 如需停止测试，可按 Ctrl+C
